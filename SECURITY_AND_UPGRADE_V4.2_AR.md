# Orbit HR v4.2.0 — الأمان والترقية

## أول تشغيل

1. شغّل `server.py` أو `start_server.py`.
2. ينشئ النظام قاعدة البيانات داخل `server_data`.
3. تُنشأ كلمات مرور مؤقتة قوية داخل الملف المحلي:
   `server_data/INITIAL_ADMIN_CREDENTIALS.txt`
4. افتح الملف على جهاز السيرفر فقط، وسجّل الدخول، ثم غيّر كلمة المرور فورًا.
5. لا ترفع مجلد `server_data` إلى استضافة ملفات عامة، ولا ترسله لأي مستخدم.

يمكن تحديد كلمة مرور مسؤول النظام قبل أول تشغيل بواسطة متغير البيئة:

```bash
ORBIT_HR_ADMIN_PASSWORD="StrongPassword123" python server.py
```

## أهم تغييرات v4.2.0

- حظر قراءة قاعدة البيانات، مفاتيح الجلسات، ملفات المصدر والنسخ الاحتياطية من الويب.
- منع Path Traversal وقوائم المجلدات.
- حذف كلمات المرور النصية من `initial_state.json` وحالة الواجهة وLocal Storage.
- إلغاء تسجيل الدخول المحلي عند تعذر السيرفر.
- صلاحيات دقيقة من السيرفر بدل منح المدير وصولًا عامًا.
- مزامنة محمية من التعارضات بدل استبدال حالة البرنامج بصمت.
- تسجيل الحضور بوقت السيرفر وتوقيت `Africa/Cairo`.
- عدم اعتبار صورة الكاميرا تعرّف وجه؛ تُحفظ كدليل يحتاج مراجعة.
- فحص دقة GPS وإشارة الموقع الوهمي.
- احتساب الرواتب مع تاريخ التعيين ونهاية العقد والحقول الصحيحة للتأمين والضريبة.
- احتساب الإضافي المعتمد فقط عند تفعيل سياسة الاعتماد.
- منع تداخل الإجازات والمأموريات ومنع اعتماد المستخدم لطلبه الشخصي.
- فحص نوع ومحتوى المرفقات قبل حفظها.
- النسخة الاحتياطية أصبحت ZIP تشمل SQLite والمرفقات وحالة JSON منزوعة الأسرار.

## ملاحظات إنتاجية

- ضع البرنامج خلف HTTPS Reverse Proxy.
- حدّد `ORBIT_HR_ALLOWED_ORIGINS` بدومين التطبيق بدل `*`.
- احمِ مجلد `server_data` بصلاحيات نظام التشغيل.
- استخدم قاعدة بيانات مُدارة مثل PostgreSQL عند التوسع لعدد كبير من المستخدمين.
- التعرف البيومتري الحقيقي يحتاج خدمة متخصصة، موافقات قانونية، كشف حيوية، وسياسة احتفاظ وحذف واضحة.
