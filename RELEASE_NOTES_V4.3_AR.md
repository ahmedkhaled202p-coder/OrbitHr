# Orbit HR v4.3.0-ready — ملاحظات النسخة

هذه النسخة هي تحديث جاهزية فوق v4.2.0، وتركّز على إصلاح الأعطال التي ظهرت في الاختبار العملي وتحسين أمان التشغيل.

## أهم ما تم إصلاحه

1. إصلاح تسجيل الحضور والانصراف من السيرفر:
   - وقت الحضور والانصراف أصبح يعتمد على وقت السيرفر وتوقيت `Africa/Cairo`.
   - تم إصلاح خطأ كان يفرّغ سجل الحضور عند تسجيل الخروج ويرجع السجل كـ `{}`.
   - منع تسجيل الخروج قبل تسجيل الحضور.
   - منع تكرار تسجيل الحضور أو الخروج لنفس اليوم.
   - الكاميرا محفوظة كدليل فقط ولا يتم الادعاء بوجود مطابقة وجه بيومترية.

2. أمان الملفات:
   - منع تحميل `server.py` و`initial_state.json` وقاعدة البيانات وملفات `server_data` من المتصفح.
   - منع مسارات الخروج من مجلد البرنامج Path Traversal.
   - منع تخزين طلبات `/api/*` في Service Worker cache.

3. كلمات المرور والحسابات:
   - كلمات المرور لا تُرسل للواجهة ولا تُحفظ في حالة البرنامج.
   - الحسابات الأولية تُجبر المستخدم على تغيير كلمة المرور بعد أول دخول.
   - إلغاء الدخول المحلي غير الآمن من المتصفح.

4. الصلاحيات:
   - الموظف لا يستطيع حفظ حالة النظام بالكامل.
   - المدير لا يستطيع إدارة حسابات الدخول من API المستخدمين إلا إذا كان Admin.
   - العمليات اليومية تستخدم APIs منفصلة بدل استبدال حالة النظام كلها.

5. الرواتب:
   - توحيد أكبر بين حقول الواجهة وحقول السيرفر.
   - مراعاة تاريخ التعيين وتاريخ نهاية الخدمة عند حساب الراتب.
   - إضافة مسار حالات للراتب: draft → reviewed → approved → paid → locked.

## اختبارات تم تنفيذها

- تشغيل السيرفر على نسخة نظيفة بدون قاعدة بيانات مرفقة.
- تسجيل دخول Admin / Manager / Employee.
- التأكد من عدم ظهور كلمات مرور واضحة في بيانات المستخدمين.
- تسجيل حضور موظف من API السيرفر.
- تسجيل خروج لنفس الموظف والتأكد من عدم تفريغ السجل.
- منع الموظف من `PUT /api/state`.
- حساب رواتب شهر 2026-07 من السيرفر.
- اختبار منع الوصول إلى الملفات الحساسة من المتصفح.

## ملاحظات مهمة قبل الاستخدام الحقيقي

- هذه النسخة صالحة للتجربة الداخلية على سيرفر آمن، لكنها ليست بديلًا عن بناء Android/iOS native كامل.
- رفعها على الإنترنت يتطلب HTTPS وReverse Proxy وإعداد `ORBIT_HR_ALLOWED_ORIGINS`.
- التحقق بالكاميرا هو إثبات صورة فقط، وليس تعرّف وجه حقيقي. التعرف البيومتري يحتاج خدمة منفصلة.
- يجب تغيير كلمات المرور الافتراضية فور أول تسجيل دخول.
- لا تضع بيانات موظفين حقيقية قبل ضبط النسخ الاحتياطي وسياسة الخصوصية النهائية.
