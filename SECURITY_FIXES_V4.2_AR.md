# تقرير إصلاحات Orbit HR v4.2.0-security

تاريخ التنفيذ: 2026-07-15

## الإصلاحات المنفذة

### 1. حماية الملفات وقاعدة البيانات
- منع عرض أو تحميل ملفات الكود والسيرفر وقاعدة البيانات من المتصفح.
- منع الوصول إلى `server_data` و`initial_state.json` و`server.py` وملفات النسخ الاحتياطي مباشرة.
- إغلاق مسارات Path Traversal مثل `../server.py` و`%2e%2e/server.py`.
- إضافة رؤوس حماية HTTP أساسية تشمل منع الكاش للـAPI وإضافة سياسة محتوى CSP.

### 2. كلمات المرور والمصادقة
- حذف كلمات المرور النصية من `initial_state.json` ومن حالة البرنامج التي ترجع للواجهة.
- منع إرسال كلمات المرور إلى المتصفح داخل `state` أو قائمة المستخدمين.
- إنشاء حسابات الدخول في جدول `auth_users` باستخدام Hash فقط.
- إضافة ملف بيانات دخول أولية محلي داخل `server_data/INITIAL_ADMIN_CREDENTIALS.txt` عند أول تشغيل فقط، مع إجبار تغيير كلمة المرور.
- منع تسجيل الدخول المحلي بكلمات مرور مخزنة في LocalStorage.
- إضافة حد لمحاولات تسجيل الدخول المتكررة.

### 3. الصلاحيات
- جعل حفظ حالة النظام كاملة `PUT /api/state` للأدمن فقط.
- منع المدير والموظف من رؤية كلمات مرور المستخدمين.
- منع الموظف من تسجيل حضور موظف آخر.
- منع المدير من إدارة حسابات المستخدمين إلا بصلاحية أدمن.
- إضافة مزامنة آمنة `/api/sync` للأقسام القديمة بدل استبدال حالة النظام كاملة.

### 4. الحضور والانصراف
- تسجيل الحضور والانصراف من وقت السيرفر بتوقيت `Africa/Cairo` بدل قبول وقت الجهاز.
- إضافة التحقق من دقة GPS ورفض المواقع الوهمية عند إرسالها.
- منع الانصراف قبل وجود حضور.
- منع تكرار الحضور أو الانصراف لنفس اليوم.
- تحويل الكاميرا إلى “صورة إثبات” وليست “تعرّف وجه”؛ النظام لا يدّعي وجود بصمة وجه حقيقية.
- حفظ صورة إثبات الحضور كمرفق منفصل بدل وضعها داخل حالة البرنامج.

### 5. الرواتب
- إصلاح التعامل مع توقيت القاهرة في حساب التأخير والانصراف المبكر.
- إضافة دعم `hireDate` و`contractStart` و`terminationDate` و`contractEnd` عند احتساب الشهر.
- توحيد أسماء حقول التأمين والضريبة بين الواجهة والسيرفر: `insuranceDeduction / taxDeduction` و`insurance / tax`.
- دعم حساب الشهر حتى اليوم أو الشهر كاملًا حسب اختيار المستخدم.
- حماية سجلات الرواتب المعتمدة أو المصروفة أو المغلقة من إعادة الاحتساب إلا عند استخدام Force.

### 6. الواجهة والـService Worker
- منع تخزين `/api/*` والملفات الحساسة في كاش الـService Worker.
- تنظيف الحالة قبل حفظها في LocalStorage.
- ربط عمليات الموظفين، الحضور، الإجازات، المأموريات، والرواتب بالـAPI بدل الاعتماد على التخزين المحلي فقط.
- تغيير نصوص الكاميرا إلى “إثبات الكاميرا” لتجنب الادعاء بوجود تعرّف وجه حقيقي.

## اختبارات Smoke Test
- `GET /api/health`: يعمل ويرجع الإصدار `4.2.0-security`.
- فتح `/server.py`: مرفوض/غير موجود.
- فتح `/initial_state.json`: مرفوض/غير موجود.
- فتح `/server_data/orbit_hr.sqlite3`: مرفوض/غير موجود.
- فتح مسارات `../server.py` و`%2e%2e/server.py`: مرفوضة.
- تسجيل دخول الأدمن والمدير والموظف: يعمل بكلمات البداية المحلية.
- حالة البرنامج لا تحتوي حقول Password.
- الموظف لا يستطيع حفظ حالة النظام كاملة.
- الموظف لا يستطيع تسجيل حضور موظف آخر.
- الموظف يستطيع تسجيل حضوره لنفسه من API السيرفر.
- تحميل Backup السيرفر لا يعمل إلا بتوكن مصادقة.

## ملاحظات مهمة قبل النشر الحقيقي
- الكاميرا حاليًا إثبات فقط، وليست مطابقة وجه بيومترية. لو تريد Face Recognition حقيقي يلزم إضافة محرك Face Matching وLiveness Detection.
- منع Fake GPS الكامل يحتاج جزء Native في Android/iOS بجانب فحص السيرفر.
- بعض الأقسام القديمة ما زالت تتم مزامنتها عبر `/api/sync` بدل API مستقل لكل شاشة؛ هذا أفضل من حفظ الحالة كاملة لكنه ليس التصميم النهائي المثالي.
- سياسة الخصوصية وشروط الاستخدام ما زالت تحتاج مراجعة قانونية قبل Google Play وApp Store.
